What is this API?

This API provides endpoints for managing support tickets and user accounts.

Base URL

domain.com/api

Login

POST
/auth/enter

Request Body:

{
    "username": "John Doe",
    "email": "johndoe@example.com",
    "password": "Password123"
}

Responses:

Success (200):

{
    "message": "User has been loged in",
    "status": true
}

Email not registered (400):

{
    "message": "Email aren't registered",
    "status": false
}

Incorrect password (400):

{
    "message": "Password is incorrect",
    "status": false
}

Register

POST
/auth/register

Request Body:

{
    "username": "John Doe",
    "email": "johndoe@example.com",
    "password": "Password123"
}

Responses:

Success (200):

{
    "message": "User has been registered",
    "status": true
}

Email already exists (400):

{
    "message": "Email already exists",
    "status": false
}

Invalid password (400):

{
    "message": "Password is invalid",
    "status": false
}

Fetch User Information

GET
/user/fetch

Description: Fetches information about a specific user.

Query Parameters:

  • email: User email (example: email=johndoe@example.com)

Responses:

Success (200):

{
    "username": "John Doe",
    "email": "johndoe@example.com",
    "password": "********",
    "created_at": "2023-07-15T14:30:45Z",
    "status": true
}

User not found (404):

{
    "message": "User not found",
    "status": false
}

List All Users

GET
/user/fetchall

Description: Fetches all users from the system.

Responses:

Success (200):

{
    "users": [
        {
            "username": "John Doe",
            "email": "johndoe@example.com"
        },
        {
            "username": "Jane Smith",
            "email": "janesmith@example.com"
        }
    ],
    "status": true
}

    "callL_label": "Room 19",
    "call_equipment": "Computer 97",
    "call_explain": "Computer has downed of the table"
}

Responses:

Success (200):

{

    "status": true
}

}

Fetch Tickets

GET
/called/fetch

Query Parameters:

  • author: Email of the ticket author (example: author=johndoe@example.com)
  • status: Status filter (possible values: all, pending, doing, conclued)

Examples:

  • List all tickets: /called/fetch?author=johndoe@example.com&status=all
  • List pending tickets: /called/fetch?author=johndoe@example.com&status=pending

Responses:

Success (200):

{
    "calls": [
        {
            "call_id": "call_987654321",
            "call_name": "Broken computer",
            "call_status": "doing"
        },
        {
            "call_id": "call_987654322",
            "call_name": "Printer not working",
            "call_status": "pending"
        }
    ],
    "status": true
}

No tickets (404):

{
    "message": "Author don't have calls",
    "status": false
}
